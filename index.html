Coreos

How to setup a CoreOS cluster along with some useful bells and whistles

Download .zip Download .tar.gz View on GitHub

Summary

  • Setup 3 node CoreOS cluster via Cloud Formation fronted by an ELB
  • Deploy monitoring and logging containers
  • Deploy http server container
  • Setup .bit domain
  • Add CI/CD

1) Setup Docker

install virtualbox (needed by boot2docker): https://www.virtualbox.org/

install vagrant (not strictly necessary but comes in handy if you want to create local clusters vs EC2): https://www.vagrantup.com/downloads

install docker locally (assuming you are on a mac here so youâ€™ll need use boot2docker): https://docs.docker.com/installation/mac/

create an account on dockerhub: https://hub.docker.com/

2) Setup AWS

Docs: http://docs.aws.amazon.com/cli/latest/userguide/cli-chap-getting-set-up.html

Get an account with Amazon AWS. You will have to pay for your usage. A single CoreOS cluster with an ELB is costing me about $3 a day. Register here: https://aws.amazon.com/ Add an account via IAM Generate an SSH keypair and upload the public key

Install the AWS command line tools:

wget https://s3.amazonaws.com/aws-cli/awscli-bundle.zip
unzip awscli-bundle.zip
sudo ./awscli-bundle/install -i /usr/local/aws -b /usr/local/bin/aws

Add necessary environment variables to your ~/.bash_profile:

# AWS creds
export AWS_ACCESS_KEY_ID=XXXXXXXXXXXXXX
export AWS_SECRET_ACCESS_KEY=XXXXXXXXXXXXXXXXXXXXXX
export AWS_DEFAULT_REGION=us-west-1
export AWS_DEFAULT_OUTPUT=text
